##機能要件（CLI版　）

1. **ユーザー認証機能**
    * コマンド実行時の引数または対話型プロンプトによるログインIDとパスワードの照合。
2. **タスク管理機能 (CRUD)**
    * 課題のタイトル、科目、締切日、メモの登録・編集・表示・削除。
3. **ステータス管理機能**
    * 各課題に対して「未着手」「進行中」「完了」のいずれかを設定・変更できる機能。
4. **締切リマインド機能**
    * 一覧表示時、締切日が当日または翌日のタスクをターミナル上で色を変えて強調表示。
5. **フィルタリング・検索機能**
    * 「科目名」や「ステータス」を指定した絞り込み表示、およびキーワード検索。

# 非機能要件 (CLI版)

1. **レスポンス性能**
    * コマンド実行から結果が表示されるまで、1秒以内に処理を完了すること。
2. **データの安全性**
    * サーバー側のデータベース (MariaDB) を使用し、アプリ終了後もデータが保持されること。
3. **ユーザビリティ (CLI)**
    * `--help` オプションにより、コマンドの使い方が日本語で分かりやすく表示されること。
4. **保守性**
    * Pythonの標準的なモジュール（argparse等）を使用し、将来の機能追加が容易な構造にすること。

# 表示モード (コマンド体系)

1. **ダッシュボード (Listモード)**
    * list コマンド：全タスクの進捗を一覧形式でターミナルに出力する画面。
2. **登録・編集 (Inputモード)**
    * add / edit コマンド：対話形式で詳細情報を入力、または引数で一括設定する画面。
3. **アーカイブ (Archiveモード)**
    * history または list --done：完了済みの課題のみを表示し、振り返るための画面。
