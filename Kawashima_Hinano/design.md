# 設計書

## 画面設計

### 1. 時間割一覧画面

* 曜日ごとの時間割を一覧で表示する
* 各授業の科目名・教室を表示する
* 「追加」ボタンで授業登録画面へ遷移する
* 「共有」ボタンで共有画面へ遷移する



### 2. 授業登録・編集画面

* 曜日・時限・科目名を入力できる
* 教室・担当教員を入力できる
* 「保存」ボタンで登録・更新する
* 「削除」ボタンで授業を削除する



### 3. 共有画面

* 自分の時間割を友達に送信できる
* 共有リンクを発行できる
* 友達の時間割を閲覧できる



## データ設計（3つ）

1. 授業データ

   * 科目名
   * 曜日
   * 時限
   * 教室
   * 担当教員

2. ユーザーデータ

   * ユーザー名
   * ID

3. 共有データ

   * 共有ID
   * 送信者ID
   * 受信者ID



## 処理の流れ（1つ）

1. ユーザーが授業を登録する
2. データが保存される
3. 一覧画面に時間割が表示される
4. 共有ボタンを押す
5. 共有リンクが生成される
6. 友達がリンクから時間割を閲覧する



## システム構成

* フロントエンド：画面表示・入力処理
* バックエンド：データの保存・共有処理
* データベース：授業・ユーザー・共有情報の管理
